## 工作流1：需求工程自动化流水线

**文件**：`n8n/workflow-01-需求工程流水线.json`

**流程节点**（8步自动化）：

```
手动触发 → A1涉众Agent需求获取 → A2需求质量检测 → IF有质量问题?
                                                          ├─ 是 → 回退涉众Agent → 回到A1
                                                          └─ 否 → A3 UML建模 → A4 SRS生成
                                                                     → A5交叉验证 → IF通过?
                                                                         ├─ 是 → A6创立基线 → CCB审批 → 通知完成
                                                                         └─ 否 → 回到修复
```

| 节点 | 说明 |
|------|------|
| A1 涉众Agent需求获取 | 启动5个涉众Agent并行对话，输出到 raw/dialogs/ 和 raw/notes/ |
| A2 需求质量检测 | 检测模糊/不一致/矛盾/缺失四类问题 |
| 回退节点 | 发现问题→回退至对应涉众重新获取，生成回退记录 |
| A3 UML建模 | 生成用例图/类图/时序图/3张活动图 |
| A4 SRS生成 | 基于IEEE 830标准生成SRS初稿 |
| A5 交叉验证 | 四维度验证（完备性/一致性/可追溯性/UML对齐） |
| A6 创立基线 | 生成基线文件（SRS正式版+基线说明+溯源矩阵+需求清单） |
| CCB审批 | 编制→审核→批准三级签批，进入软件设计阶段 |

## 如何安装 n8n 并导入工作流

### 1. 安装 n8n

```bash
npm install n8n -g
```

### 2. 启动 n8n

```bash
n8n start
```

启动后浏览器访问 `http://localhost:5678`

### 3. 导入工作流

1. 登录 n8n → 右上角「Import from File」
2. 选择 `n8n/workflow-01-需求工程流水线.json`
3. 导入后可看到完整的可视化流程

### 4. 配置 Webhook URL

工作流包含 Webhook 节点，启动后自动获得访问地址，可从外部系统调用触发流程。
