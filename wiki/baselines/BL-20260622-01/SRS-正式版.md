## 1. 引言

### 1.1 项目背景

某中型教育培训机构（以下简称"机构"）拥有15年办学历史，目前在深圳设有3个校区（福田总部、南山校区、宝安校区），业务涵盖K12学科辅导、艺术培训和语言培训三大品类，在册学员约3000人，教师团队200余人。随着规模扩大，传统的纸质记录和Excel表格管理方式已无法满足日益复杂的教务和收费管理需求，亟需建设一套专业的教育培训机构教务收费管理系统。

### 1.2 项目目标

1. **教务流程标准化**：实现学员信息、课程、班级、排课、考勤的统一管理
2. **收费管理精细化**：覆盖多种收费模式、优惠政策、退费处理、欠费催缴的全流程管理
3. **数据统计自动化**：提供多维度营收报表、学员分析、教师绩效统计，支撑经营决策
4. **多校区协同管理**：支持多校区数据隔离与总部汇总的双重管理模式
5. **移动端服务体验**：为家长和教师提供便捷的移动端应用（课表、缴费、签到）

### 1.3 文档范围

本SRS文档定义了"教育培训机构教务收费管理系统"的完整功能需求和非功能需求，覆盖8大功能模块共100条功能需求和29条非功能需求，作为后续设计、开发和测试的权威依据。

### 1.4 术语定义

| 术语 | 定义 |
|------|------|
| 课时包 | 预购固定数量课时的收费模式（如10节/20节/50节） |
| 阶梯定价 | 购买课时数量越多单价越低的定价策略 |
| 涉众 | 与系统有利益关系的角色（管理员、教务、财务、教师、家长） |

| 补签 | 超过规定时间后补录的签到记录 |
| RBAC | Role-Based Access Control，基于角色的访问控制 |

## 2. 总体描述

### 2.1 产品视角

本系统是一个面向教育培训机构的综合教务收费管理平台，采用B/S架构，包含PC管理后台（供管理员、教务、财务使用）和移动端应用（供教师、学生/家长使用）。系统支持多校区部署，各校区数据逻辑隔离，总部可跨校区汇总查看。

### 2.2 用户角色

| 角色 | 职责概述 | 主要使用终端 | 优先级 |
|------|---------|------------|--------|
| 机构管理员 | 系统全局配置、权限管理、数据报表分析、跨校区管理 | PC | P0 |
| 教务人员 | 学员管理、课程管理、班级管理、排课、考勤统计、教师管理 | PC为主，移动端辅助 | P0 |
| 财务人员 | 收费标准设置、收费管理、退费处理、欠费催缴、财务报表 | PC | P0 |
| 教师 | 课表查看、学员签到、课时统计查看、学员信息查看 | 移动端为主 | P1 |
| 学生/家长 | 课程浏览、在线缴费、课表查看、请假申请、课时余额查看 | 移动端为主 | P1 |

### 2.3 系统模块架构

```
教育培训机构教务收费管理系统
├── 学员管理 (Student Management)
├── 课程管理 (Course Management)
├── 班级管理 (Class Management)
├── 教师管理 (Teacher Management)
├── 收费管理 (Fee Management)
├── 考勤管理 (Attendance Management)
├── 报表统计 (Report & Statistics)
└── 系统管理 (System Administration)
```

### 2.4 核心业务流程

```
学员报名 → 选课分班 → 费用计算 → 缴费确认 → 排课开班 → 日常考勤
                                                          ↓
              ← 续费提醒 ← 课时消耗统计 ← 退费处理 ← 学员异动
```

### 2.5 假设与依赖

| 编号 | 假设/依赖 | 说明 |
|------|----------|------|
| AS-001 | 机构拥有稳定的服务器环境 | 系统部署在符合等保二级的服务器上 |
| AS-002 | 家长具备智能手机和基本操作能力 | 移动端面向普通消费者 |
| AS-003 | 第三方支付平台可用 | 微信支付和支付宝接口正常可用 |
| AS-004 | 历史数据可迁移 | 现有Excel数据可导入系统 |

| CON-018 | 课酬计算规则需由财务模块配置，教师端仅展示计算结果的只读视图 | 教师 |
| CON-019 | 排课数据需与教室资源管理模块联动，教室冲突和教师冲突需同时检测 | 教务人员 |
| CON-020 | 学员状态变更需触发财务模块的费用重算流程 | 教务人员 |
