### 1.2 基线目的

BL-20260622-01是"教育培训机构教务收费管理系统"项目的首个正式需求基线，旨在将经过交叉验证的需求规格文档进行正式冻结和版本化管理，作为后续设计阶段、开发阶段和测试阶段的权威依据。

### 1.3 基线范围

本基线覆盖项目的全部功能需求和非功能需求，包括：

- **功能需求**：8个功能模块、100条功能需求（覆盖5个涉众角色）
- **非功能需求**：29条非功能需求（覆盖性能、安全性、可靠性、准确性、易用性、兼容性等维度）
- **UML模型**：6份UML建模产物（用例图、类图、时序图、活动图）
- **需求验证**：1份需求验证报告 + 1份缺陷分析报告

### 3.2 变更管理规则

自本基线创建起，所有对基线文档的修改需遵循以下变更管理流程：

1. **变更申请**：提交变更请求（CR），说明变更原因、变更内容和影响范围
2. **变更评审**：由需求管理委员会评估变更的必要性和可行性
3. **变更批准**：获得批准后方可执行变更
4. **变更实施**：更新相关基线文档，记录变更内容和版本号
5. **基线发布**：变更后的文档集作为新的基线版本发布（如 BL-20260622-02）

## 6. 基线用途

### 6.1 后续阶段指引

本基线文档将作为以下阶段活动的权威输入：

| 阶段 | 活动 | 使用的基线文档 |
|------|------|--------------|
| 设计阶段 | 架构设计、数据库设计、接口设计 | SRS-正式版、需求清单、UML类图 |
| 设计阶段 | ADR决策记录编写 | SRS-正式版、缺陷分析报告 |
| 开发阶段 | 功能实现、单元测试编写 | 需求清单、溯源矩阵 |
| 测试阶段 | 测试用例设计、验收测试 | SRS-正式版、需求清单 |
| 部署阶段 | 用户验收标准定义 | SRS-正式版（全部功能+非功能需求） |

### 6.2 需求编号体系

本基线采用以下需求编号体系，后续所有需求跟踪和引用均以此编号为准：

```
功能需求：REQ-{角色代码}-{序号}
非功能需求：NF-{角色代码}-{序号}
缺陷编号：DEF-{序号}
问题编号：ISS-{序号}

角色代码：
  ADMIN = 机构管理员
  FIN   = 财务人员
  EDU   = 教务人员
  TCH   = 教师
  STU   = 学生/家长
```
