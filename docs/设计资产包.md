# 设计资产包（Design Asset Package）

> **项目**: 教育培训机构教务收费管理系统（EduFeeMS）
> **版本**: V1.0
> **编制日期**: 2026-06-22
> **编制人**: AI Assistant

---

## 概述

本文档定义了EduFeeMS项目的完整设计资产包，采用五层结构组织，涵盖代码资产、接口契约资产、知识图谱资产、子技能资产和提示词模板资产。设计资产包的目的是确保项目设计知识的结构化沉淀和高效复用，使新团队成员能够快速理解系统架构，使AI辅助开发工具能够基于结构化契约和约束产生更高质量的代码。

资产包的五层结构对应"从具体到抽象、从实现到指导"的知识层次递进关系，低层资产（代码、接口）是具体的实现产物，高层资产（知识图谱、提示词模板）是抽象的设计指导，子技能资产则作为可复用的分析验证能力贯穿各层。

### 1.4 代码复用指南

**后端代码复用模式**：

1. **BaseEntity继承复用**：所有Entity类继承`BaseEntity`，自动获得id、createTime、updateTime、createBy、updateBy、isDeleted等通用字段和自动填充能力
2. **统一响应格式**：所有Controller返回`R<T>`类型，提供code/message/data标准结构
3. **全局异常处理**：所有异常统一由`GlobalExceptionHandler`捕获和转换，业务异常使用`BusinessException`
4. **分页查询模式**：统一使用MyBatis-Plus的`IPage<T>`和`Page<T>`，Controller层接收`PageQuery`基类参数
5. **模块间调用**：通过Service接口注入，禁止直接依赖实现类或跨层调用Repository

**前端代码复用模式**：

1. **API封装层**：所有后端调用统一通过`api/`目录下的模块文件，使用`request.js`中的Axios实例（含JWT拦截器）
2. **状态管理**：全局状态（用户信息、权限、配置）使用Pinia Store，页面级状态使用组件本地ref/reactive
3. **组件层级**：页面(Page) → 业务组件(Component) → 基础组件(BaseComponent)

### 2.2 接口清单

契约共定义21个API接口，按业务域分为6组：

| 业务域 | 接口数 | 主要接口 |
|--------|--------|----------|
| auth | 3 | POST /auth/login, POST /auth/logout, GET /auth/me |
| students | 7 | GET/POST /students, GET/PUT /students/{id}, POST /students/{id}/enroll, POST /students/{id}/transfer, POST /students/{id}/withdraw |
| courses | 2 | GET/POST /courses |
| classes | 3 | GET/POST /classes, POST /classes/{id}/schedules |
| fees | 4 | POST /fees/payment, POST /fees/refund, GET /fees/records, GET /fees/dunning |
| attendances | 3 | POST /attendances/checkin, POST /attendances/leave, GET /attendances/statistics |

### 2.3 接口调用说明

**认证机制**：
- 所有接口（除`/auth/login`外）需在Header中携带`Authorization: Bearer {token}`
- JWT Token有效期为2小时，支持刷新

**统一响应格式**：
```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```

**分页查询模式**：
- 请求参数：`pageNum`（页码，默认1）、`pageSize`（每页条数，默认10）
- 响应data结构：`{ pageInfo: { pageNum, pageSize, total, pages }, records: [...] }`

**错误响应**：
- 400：参数校验失败（ValidationErrorResponse含field级错误信息）
- 401：未登录或Token过期
- 403：权限不足
- 404：资源不存在
- 500：服务器内部错误

## 复用指南

### 在新项目中迁移本资产包

#### 步骤1：评估适配性

在将本资产包复用到新项目时，首先评估以下维度：

| 评估维度 | 适配条件 | 调整幅度 |
|----------|----------|----------|
| 业务领域 | 教育培训/教务收费 | 核心业务逻辑需重写 |
| 技术栈 | Java Spring Boot + Vue 3 | 架构设计可直接复用 |
| 模块划分 | 9模块分层架构 | 根据需求重新规划模块 |
| 数据库 | MySQL + MyBatis-Plus | 表结构需重新设计 |
| OpenAPI | 3.0.3 | 接口定义需重新编写 |

#### 步骤2：迁移核心资产

推荐迁移顺序（按依赖关系从底向上）：

1. **知识图谱资产（第三层）**：
   - 复制ASD（架构风格声明）并调整架构规则
   - 复制MDS（模块划分方案）模板并重新定义模块
   - 复制DTS（模块依赖拓扑）模板并重新分析依赖

2. **接口契约资产（第二层）**：
   - 复制OpenAPI契约模板（components/schemas中的通用结构）
   - 重新定义业务接口的paths

3. **代码资产（第一层）**：
   - 复制edu-common模块（BaseEntity、R类、异常处理、分页）
   - 复制edu-auth模块的认证框架（JWT、RBAC）
   - 复用约束配置（TLCD三层约束）

4. **子技能资产（第四层）**：
   - 5个子技能SKILL.md均可直接复用
   - 仅需调整技能中引用的模块名和REQ编号

5. **提示词模板资产（第五层）**：
   - 代码生成提示词模板的架构约束部分直接复用
