### 回退 #3：优惠叠加规则冲突（财务人员 + 教务人员）

| 字段 | 内容 |
|------|------|
| **回退编号** | RB-003 |
| **发现人** | A2-需求分析智能体 |

| **状态** | ✅ 已解决 |

### 回退 #4：数据安全 vs 操作便捷性冲突（财务人员 vs 教师）

| 字段 | 内容 |
|------|------|
| **回退编号** | RB-004 |
| **发现人** | A2-需求分析智能体 |
| **检测类型** | 矛盾需求 |
| **来源涉众** | 财务人员（A1-3）vs 教师（A1-4） |
| **冲突描述** | 财务人员要求所有金额修改需二次密码验证（安全性优先），教师要求考勤签到支持一键操作（便捷性优先）。两者在权限设计层面存在天然张力 |
| **回退时间** | 2026-06-22 |

| **纳入SRS** | REQ-ADMIN-005（RBAC权限控制）, C-ARCH-003（分层鉴权） |
| **状态** | ✅ 已解决 |

## 4. 回退流程总结

```

       ↓
A2需求分析智能体质量检测
       ↓
   ┌──发现问题──┐
   │            │
   ▼            ▼
模糊/不一致   矛盾

   ▼            ▼

   └────┬───────┘
        ▼

        ▼
   A5需求验证智能体交叉验证
```

## 5. 回退带来的SRS改进

通过11轮回退，SRS初稿质量得到显著提升：

| 改进维度 | 改进前 | 改进后 |
|----------|--------|--------|
| 退费规则细节 | "按比例退费"（模糊） | 精确到已消耗课时统计口径、退款方式、精度要求 |
| 优惠叠加规则 | "多种优惠可叠加"（冲突） | "顺序叠加模式"固定优先级链条 |
| 数据安全设计 | "密码验证"（笼统） | RBAC分级：敏感操作强制二次验证，常规操作简化 |
| 课酬计算 | "课时×单价"（简略） | 区分正常/代课/补课三种类型，分别计算 |
| 通知策略 | "及时推送"（模糊） | 时段限制8:00-21:00、多渠道去重、家长偏好配置 |
