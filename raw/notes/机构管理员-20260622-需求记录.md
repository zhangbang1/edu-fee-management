# 机构管理员 — 需求记录

**涉众角色**：机构管理员（创始人/管理者）
**记录日期**：2026-06-22
**优先级**：P0（最高优先级）
**需求编号前缀**：REQ-ADMIN

---

## 一、功能需求

### 1.1 多校区管理

| 编号 | 需求描述 | 优先级 | 业务价值 |
|------|----------|--------|----------|
| REQ-ADMIN-001 | 系统需支持多校区架构，管理员可创建、编辑、停用校区信息（名称、地址、联系电话、负责人等） | P0 | 多校区运营的基础支撑 |
| REQ-ADMIN-002 | 各校区数据独立存储，支持按校区维度隔离查询和管理 | P0 | 数据隔离保障各校区独立核算 |
| REQ-ADMIN-003 | 总部管理员可跨校区查看汇总数据，包括营收、学员数、教师数、开班数等核心指标 | P0 | 支撑总部层面的经营决策 |
| REQ-ADMIN-004 | 支持将一个校区的学员或教师调配到另一个校区，并自动同步相关费用和课程信息 | P1 | 校区资源灵活调配 |

### 1.2 权限管理

| 编号 | 需求描述 | 优先级 | 业务价值 |
|------|----------|--------|----------|
| REQ-ADMIN-005 | 系统需实现基于角色（RBAC）的权限控制体系，预置管理员、教务、财务、教师、学生/家长五种角色 | P0 | 保证不同用户只能访问授权范围内的数据和功能 |
| REQ-ADMIN-006 | 管理员可创建自定义角色，灵活配置功能权限和数据权限（按校区、按课程品类等） | P1 | 适配机构个性化管理需求 |
| REQ-ADMIN-007 | 管理员可创建、禁用、删除系统用户账号，关联角色和校区 | P0 | 用户生命周期管理 |
| REQ-ADMIN-008 | 所有关键操作（金额修改、权限变更、数据删除等）需记录操作日志（操作人、时间、IP、内容） | P0 | 审计追溯和内部风控 |

### 1.3 数据报表与统计分析

| 编号 | 需求描述 | 优先级 | 业务价值 |
|------|----------|--------|----------|
| REQ-ADMIN-009 | 系统需提供营收统计报表，支持按日/周/月/季度/年度查看，支持按校区、课程品类、收费类型等维度筛选 | P0 | 实时掌握机构经营状况 |
| REQ-ADMIN-010 | 系统需提供学员分析报表，包括新生报名趋势、在读学员数、退学率、续费率等指标 | P0 | 分析招生质量和学员留存情况 |
| REQ-ADMIN-011 | 系统需提供教师分析报表，包括教师课时统计、课酬汇总、学员评价、出勤率等 | P1 | 教师绩效管理和课酬核算 |
| REQ-ADMIN-012 | 系统需提供课程分析报表，包括各课程报名人数、满座率、退费率、营收贡献等 | P1 | 课程产品优化决策 |
| REQ-ADMIN-013 | 所有报表支持图形化展示（柱状图、折线图、饼图等）和一键导出（Excel/PDF） | P0 | 提升数据可读性和汇报效率 |

### 1.4 系统配置与管理

| 编号 | 需求描述 | 优先级 | 业务价值 |
|------|----------|--------|----------|
| REQ-ADMIN-014 | 管理员可配置机构基础信息（名称、Logo、地址、联系方式、社会信用代码等），信息在收据和系统中自动展示 | P0 | 机构品牌形象统一 |
| REQ-ADMIN-015 | 系统需支持学期/学段设置，管理员可设定学期起止日期，系统据此管理班级和课程周期 | P0 | 教学周期的结构化管理 |
| REQ-ADMIN-016 | 系统需支持数据备份与恢复功能，支持自动定时备份（每日/每周）和手动备份，备份数据加密存储 | P0 | 防止数据丢失，保障业务连续性 |
| REQ-ADMIN-017 | 系统需支持通知模板配置，管理员可设置各类通知（缴费提醒、课时不足、调课通知等）的推送模板和渠道 | P1 | 提升沟通效率和品牌一致性 |
| REQ-ADMIN-018 | 系统需支持操作日志查询，可按时间、操作人、操作类型、操作对象等多条件筛选 | P1 | 内部审计和异常操作追踪 |

## 二、非功能需求

| 编号 | 需求描述 | 类型 | 优先级 |
|------|----------|------|--------|
| NF-ADMIN-001 | 系统需满足7×24小时高可用，核心功能可用性不低于99.9%，学期报名高峰期需支撑500以上并发用户 | 可靠性 | P0 |
| NF-ADMIN-002 | 所有学员个人敏感信息（姓名、证件号、联系方式）需加密存储（AES-256），传输过程使用HTTPS/TLS加密 | 安全性 | P0 |
| NF-ADMIN-003 | 财务相关数据需实现数据库层面的加密，备份文件同样加密 | 安全性 | P0 |
| NF-ADMIN-004 | 报表查询响应时间不超过3秒，大数据量导出不超过30秒 | 性能 | P1 |
| NF-ADMIN-005 | 系统管理后台需支持主流浏览器（Chrome/Edge/Firefox/Safari最新两个版本） | 兼容性 | P1 |
| NF-ADMIN-006 | 系统需满足《个人信息保护法》和《数据安全法》的合规要求，提供数据删除和导出功能 | 合规性 | P0 |

---

## 三、约束条件

1. 系统需部署在符合国家网络安全等级保护二级或以上的服务器环境
2. 收据格式和退费规则需预留教育主管部门备案要求的配置项
3. 系统需支持后续与第三方系统（如企业微信、钉钉、财务软件）的对接扩展
4. 首期需覆盖PC管理后台和移动端（家长/教师端），后续可扩展微信小程序
5. 数据迁移需兼容现有Excel历史数据导入
