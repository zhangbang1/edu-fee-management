# 教师 — 需求记录

**涉众角色**：授课教师（兼职/全职教师）
**记录日期**：2026-06-22
**优先级**：P1（高优先级）
**需求编号前缀**：REQ-TCH

---

## 一、功能需求

### 1.1 课表查看

| 编号 | 需求描述 | 优先级 | 业务价值 |
|------|----------|--------|----------|
| REQ-TCH-001 | 教师可通过系统查看个人课程表，支持按日/周/月切换视图 | P0 | 教师日常教学安排的基础工具 |
| REQ-TCH-002 | 课表需显示每节课的时间、班级名称、课程名称、教室、学员人数等关键信息 | P0 | 提供完整的上课参考信息 |
| REQ-TCH-003 | 课程变更（调课、停课、代课、加课）时系统需向教师推送实时通知（App推送+短信备用） | P0 | 避免教师因未及时获知变更导致教学事故 |
| REQ-TCH-004 | 教师应能查看历史课表，支持回溯至少3个月内的课程安排 | P1 | 课时统计核对和历史参考 |

### 1.2 学员信息查看

| 编号 | 需求描述 | 优先级 | 业务价值 |
|------|----------|--------|----------|
| REQ-TCH-005 | 教师可查看所教班级的学员名单，包含学员姓名、性别、年龄、就读学校、联系方式（脱敏显示）等基本信息 | P0 | 教师了解学员背景的基础需求 |
| REQ-TCH-006 | 教师可查看每位学员的过往出勤记录（个人出勤率、请假次数、缺勤次数） | P1 | 了解学员学习习惯和态度 |
| REQ-TCH-007 | 教师可查看学员在本课程的历史学习记录（学习评语、阶段测试成绩、学习进步情况） | P1 | 因材施教和教学调整的参考 |
| REQ-TCH-008 | 教师可按姓名搜索所教班级的学员，快速定位目标学员 | P1 | 多班级场景下的快速查找 |

### 1.3 考勤登记

| 编号 | 需求描述 | 优先级 | 业务价值 |
|------|----------|--------|----------|
| REQ-TCH-009 | 教师可在上课前通过手机端为所教班级的学员进行签到登记，支持批量勾选出勤状态（出勤/迟到/请假/缺勤） | P0 | 考勤数字化的核心功能 |
| REQ-TCH-010 | 签到界面应显示该节课应到学员列表，支持一键全选"全部出勤"后单独标记异常状态 | P0 | 大幅提升签到效率 |
| REQ-TCH-011 | 教师可查看学员的请假申请（由家长提交），并进行审批（同意/拒绝），审批结果自动通知家长 | P1 | 请假流程线上化的关键环节 |
| REQ-TCH-012 | 系统应记录签到操作的提交时间，超出上课时间一定范围内的补签需标记为"补签" | P1 | 保证考勤数据的时效性和真实性 |
| REQ-TCH-013 | 考勤数据提交后24小时内允许教师修改，超过24小时需提交修改申请由教务审批 | P1 | 兼顾灵活性又防止随意篡改 |

### 1.4 课时统计与课酬

| 编号 | 需求描述 | 优先级 | 业务价值 |
|------|----------|--------|----------|
| REQ-TCH-014 | 系统需根据教师的实际授课记录和签到情况自动统计课时，按日/周/月汇总显示 | P0 | 课酬核算的基础数据 |
| REQ-TCH-015 | 系统需根据课时统计和教师课酬单价自动计算预估课酬（基本课时费+代课补贴+全勤奖等），教师可实时查看 | P0 | 课酬透明化，减少月末核对纠纷 |
| REQ-TCH-016 | 教师可查看每月已结算和待结算课时明细，可对有疑问的课时记录提交申诉 | P1 | 保障教师权益，减少对账工作量 |
| REQ-TCH-017 | 系统需区分正常课时、代课课时、补课课时，并按不同标准计算课酬 | P1 | 精确核算不同类型的劳动报酬 |

### 1.5 学员学习记录

| 编号 | 需求描述 | 优先级 | 业务价值 |
|------|----------|--------|----------|
| REQ-TCH-018 | 教师可在每节课后记录学员的课堂表现（课堂参与度、作业完成情况、阶段性评价等），支持文字和星级评分 | P1 | 建立学员学习档案，辅助个性化教学 |
| REQ-TCH-019 | 教师可查看学员的累计学习记录和进步趋势，按课程/时间段筛选 | P1 | 教学效果评估和调整的参考依据 |
| REQ-TCH-020 | 教师可选择将学习记录分享给家长（通过家长端查看），支持选择性发布 | P2 | 增强家校沟通，提升家长满意度 |

## 二、非功能需求

| 编号 | 需求描述 | 类型 | 优先级 |
|------|----------|------|--------|
| NF-TCH-001 | 教师端主要功能（课表查看、签到）需优先适配移动端，界面适配主流手机屏幕（375px-428px宽度） | 移动端适配 | P0 |
| NF-TCH-002 | 签到操作的流程需控制在3步以内（打开签到页→标记状态→提交），页面加载时间不超过1秒 | 易用性/性能 | P0 |
| NF-TCH-003 | 签到功能在弱网（2G/3G）和短暂离线场景下需保持可用，网络恢复后自动同步数据 | 离线可用性 | P1 |
| NF-TCH-004 | 调课、代课等重要通知的推送延迟不超过1分钟 | 通知及时性 | P0 |
| NF-TCH-005 | 教师端界面需简洁清晰，使用大字体和大按钮，适应当年龄跨度大的教师群体 | 易用性 | P1 |

## 三、约束条件

1. 教师仅能查看本人所教班级和学员的信息，不可跨班查看
2. 学员联系方式需脱敏显示（手机号隐藏中间4位），保护学员隐私
3. 签到数据需实时同步至教务后台，作为教务考勤统计和财务课时计算的数据源
4. 课酬计算规则需由财务模块配置，教师端仅展示计算结果的只读视图
5. 学习记录涉及学员个人评价信息，需按权限严格控制可见范围
